# V3.1 Planning Document — From Prototype to Product

## Where V3 Landed

V3 is live at `server-production-fe2e.up.railway.app`. It works: users create accounts, pick a topic, the AI finds articles, clusters them, writes stories, assembles a newsletter, and sends it via email. The core pipeline is solid.

But V3 was built for one use case: the deep professional briefing (The Morning Signal style). To serve a broad user base, the product needs to support fundamentally different newsletter styles, writing formats, and design sophistication.

## The Core Problem

Every newsletter V3 produces looks and reads the same way:
- Long lead story (~400 words)
- Medium quick hits (~100 words each)
- Short watch list items (~80 words each)
- Three fixed sections
- One plain MJML template with color swaps
- Raw HTML textarea for editing

This works for defense/energy/policy briefings. It doesn't work for:
- Someone who wants a quick 60-second digest
- Someone who wants a curated link list
- Someone who wants bullet-point roundups
- Someone who wants a casual, funny tone
- Someone who wants to add their own commentary
- Someone who wants a visually striking, magazine-style layout

## Newsletter Archetypes People Will Want

### 1. The Quick Digest
**Example:** Morning Brew, TLDR Newsletter
**Format:** 2-3 sentences per story + link to source. No deep analysis. Scannable in 60 seconds.
**What's missing:** Length control. Current minimum is 400 words for lead, 80-150 for quick hits. Need a "brief" mode that writes 2-3 sentences max.

### 2. The Deep Briefing
**Example:** The Morning Signal (what we built)
**Format:** Long lead story with analysis, medium quick hits, forward-looking watch list.
**What's missing:** Already works. Minor polish needed.

### 3. The Curated Link List
**Example:** Hacker News Daily, Benedict's Newsletter
**Format:** No prose. Just: headline → one-line description → link. 15-20 items.
**What's missing:** A "link list" writing mode that skips the story writer entirely and just formats the clustered articles as a list.

### 4. The Industry Roundup
**Example:** CB Insights, Stratechery
**Format:** Bullet points. "• Company X raised $50M (link) • New regulation Y (link)." Organized by category.
**What's missing:** Bullet-point formatting in the writing prompts. Current prompts produce paragraphs only.

### 5. The Personal Take
**Example:** Matt Levine's Money Stuff, Platformer
**Format:** Curated news + publisher's own commentary woven in. "Here's what happened, and here's what I think about it."
**What's missing:** A publisher commentary field per story or per section. Space for the human voice alongside the AI content.

### 6. The Team Briefing
**Example:** Internal company newsletters
**Format:** Short, actionable, maybe with action items or decisions needed. Professional but direct.
**What's missing:** Tone control (more direct/actionable), possibly an "action items" section type.

### 7. The Niche Hobbyist
**Example:** Skateboarding news, fishing reports, cooking trends
**Format:** Casual, fun, community-oriented. Emojis welcome. Informal tone.
**What's missing:** Tone/style control. Current writing is always formal/professional.

## The Big Gaps

### Gap 1: Writing Style & Length Control

Users need to choose how the AI writes. This is a per-profile setting.

**Length presets:**
- **Brief** — 2-3 sentences per story + source link. Lead story: 50-100 words. Quick hits: 20-40 words.
- **Standard** — Current behavior. Lead: 400+ words. Quick hits: 80-150 words.
- **Detailed** — Even longer. Lead: 600+ words with multiple "Why This Matters" angles.
- **Link only** — No prose. Headline + one-line description + URL.

**Style presets:**
- **Professional** — Current default. Formal, analytical, third-person.
- **Conversational** — Casual, first-person-ish, approachable. Like talking to a smart friend.
- **Punchy** — Short sentences. Active voice. No filler. Like a wire service.
- **Analytical** — Data-focused, cause-and-effect, "what this means" framing.
- **Humorous** — Light, witty, pop culture references welcome.

**Format options:**
- **Paragraphs** — Current default.
- **Bullets** — Each story as a bullet point with sub-bullets for details.
- **Headlines + blurbs** — Headline, 1-2 sentence blurb, link. No full paragraphs.
- **Mixed** — Lead story as paragraphs, quick hits as bullets.

**Implementation:** These become template variables injected into the writing prompts. The prompt says "Write in a {{style}} tone. Each story should be {{length}} format." The output format instructions change accordingly.

### Gap 2: Rich Text Editing (WYSIWYG)

Phase 2 currently shows raw HTML in a textarea. Users need to edit text that looks like the final newsletter.

**What's needed:**
- A rich text editor (like TipTap, Slate, or Quill) that renders bold, italic, links, bullets, headings
- The editor shows formatted text, not HTML tags
- Changes save back as HTML for the assembler
- Inline image placement
- Publisher can add their own text blocks between AI-generated content

**Packages to evaluate:**
- **TipTap** — Modern, extensible, React-native. Best option for our use case.
- **Slate** — Powerful but complex. More control, more work.
- **React-Quill** — Simple but limited. Might not handle our needs.

### Gap 3: Template & Design System

Current newsletters are plain — one MJML template with color swaps. For broad appeal, we need:

**Template library:**
- **Classic Briefing** — Current template. Header, sections, footer.
- **Magazine Style** — Hero image, two-column layout for quick hits, pull quotes.
- **Minimal** — Clean, lots of whitespace, single column, elegant typography.
- **Link Digest** — Compact, no images, just a clean list of links with descriptions.
- **Bold & Modern** — Large headlines, accent colors, card-based layout.
- **Corporate** — Conservative, logo-heavy, suitable for internal distribution.

**Design capabilities needed:**
- Multiple MJML templates the user can choose from
- Per-template customization (colors, fonts, spacing)
- Custom header/banner images
- Section divider styles (lines, spacing, icons)
- Card-based layouts for quick hits (each story in its own card)
- Two-column layouts for certain sections
- Pull quotes / highlighted text blocks
- Custom CSS injection for advanced users

**Packages to evaluate:**
- **MJML** — Keep as the compilation layer. It handles email client compatibility.
- **React Email** — Alternative to MJML. React components that compile to email HTML. More developer-friendly.
- **Maizzle** — Tailwind CSS for email. More design flexibility than MJML.
- **Unlayer** — Drag-and-drop email builder. Could be embedded for visual template editing.

### Gap 4: Flexible Section Structure

Not everyone wants Lead + Quick Hits + Watch List. Options:

- **Single section** — Just one list of stories, all treated equally.
- **Two sections** — Main stories + brief mentions.
- **Three sections** — Current default (lead + briefing + watch).
- **Custom sections** — User defines their own section types with custom names and writing rules.
- **No sections** — Flat list, no grouping. Good for link digests.

### Gap 5: Publisher Voice

Space for the human to add their own words:

- **Newsletter intro** — A personal note at the top ("Hey readers, big week in AI...")
- **Per-story commentary** — Publisher's take on each story, shown alongside or after the AI content
- **Newsletter outro** — Closing thoughts, calls to action, personal updates
- **Sidebar/callout boxes** — Highlighted text blocks for opinions, recommendations, or ads

## V3.1 Priority Stack

### Must-Have (makes the product broadly useful)
1. **Writing style/length/format presets** — Per-profile settings that change how the AI writes
2. **Rich text editor** — Replace raw HTML textarea with WYSIWYG (TipTap)
3. **Template library** — 4-6 visually distinct newsletter templates
4. **Flexible sections** — Allow 1-5 sections with custom types

### Strong Second Wave
5. **Publisher commentary** — Intro/outro text fields + per-story commentary
6. **Bullet-point format** — Writing prompts that produce bullet lists
7. **Link-only mode** — Skip story writing, just format articles as a link list
8. **Two-column layouts** — For quick hits and link lists

### Later
9. **Drag-and-drop template editor** — Visual template builder (Unlayer or custom)
10. **Custom CSS** — For power users who want pixel-perfect control
11. **Template marketplace** — Users share/sell templates
12. **A/B testing** — Test different subject lines or layouts

## Technical Approach for V3.1

### New repo: `morning-signal-v3.1`
Fork from V3. V3 stays live and stable. V3.1 is the development branch.

### New packages needed:
- **TipTap** (or Slate) — Rich text editor for Phase 2
- **React Email** or keep **MJML** — Template compilation
- Possibly **Unlayer** — If we go the drag-and-drop route later

### Database changes:
- `newsletter_profiles` gets: `writing_style TEXT`, `writing_length TEXT`, `writing_format TEXT`, `template_id TEXT`
- New `newsletter_templates` table: `id`, `name`, `description`, `mjml_content`, `preview_image_url`, `is_system`, `created_by`
- `written_sections` gets: `publisher_commentary TEXT`

### Prompt changes:
- Writing prompts become parameterized: `{{style}}`, `{{lengthInstructions}}`, `{{formatInstructions}}`
- New prompt variants for each format (bullets, link-only, brief)
- Style instructions appended to existing prompts (additive, not rewrite)

### Frontend changes:
- Phase 2: Replace textarea with TipTap editor
- Profile wizard: Add style/length/format selectors
- Phase 3: Template picker with visual previews
- New: Publisher intro/outro text fields

## Questions to Answer Before Building

1. **TipTap vs Slate vs React-Quill?** TipTap is the recommendation — modern, well-documented, good React support.
2. **Keep MJML or switch to React Email?** MJML works. React Email is newer and more flexible. Could migrate gradually.
3. **How many templates at launch?** 4-6 is enough. Classic, Magazine, Minimal, Link Digest, Bold, Corporate.
4. **Should templates be user-editable?** Not at first. Preset templates with color/font customization. Full editor later.
5. **How does the link-only mode work?** Skip the story writer entirely. The clustering output becomes the newsletter content directly, formatted as a link list.
